# Phase P2b ‚Äî `internal/expr` Restricted Expression Engine (parity, behind flag)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (or superpowers:test-driven-development per task). Steps use `- [ ]` checkboxes. No production code before a red test.

**Goal:** Build a stdlib-only restricted-grammar expression engine (`internal/expr`) that resolves n8n-style `{{ ‚Ä¶ }}` templates over the P2a items model, wired into the worker as a **parity alternative** to `substituteVars` behind a flag ‚Äî NOT the sole execution channel. Security-critical: the cross-tenant scoping (S-2) and secret-channel (S-3) invariants must be correct from the first commit.

**Architecture:** Three leaf-first layers ‚Äî (1) pure engine `internal/expr` (no DB/worker import): tokenizer ‚Üí parser ‚Üí evaluator for the restricted grammar; `Resolve(template, Context) (string, error)`, `Context{Self []Item, NodeByID func(id)([]Item,error), ItemIdx int}`, with `expr.Item` mirroring `worker.Item` so the package stays a leaf. (2) worker-side scoped `NodeByID` resolver (`exprNodeResolver`) enforcing the **S-2 contract** ‚Äî resolves only ids in the executing todo's DIRECT `dependsOn` set, scoped by `project_id` (rejects out-of-set + cross-project), reusing P2a `itemsForDep`. (3) behind-flag parity wiring (`Config.ExprParity bool`): computes the expr result alongside `substituteVars`, asserts/logs equivalence, NEVER feeds expr downstream (cut-over is P3). The secret channel is untouched: secrets resolve ONCE (worker.go ~1845, `secretRefRe` + `Secrets.Resolve`, orgID from `OrgIDForProject(c.projectID)`) BEFORE either `{{name}}` channel; the expr grammar treats `secret:` as a non-matching literal emitted verbatim.

**Tech Stack:** Go stdlib only inside `internal/expr` (NO goja/starlark/JS VM). `internal/scriptengine` (Starlark) is the SEPARATE escape-hatch; P2b does not import/touch it (S-6 re-asserted). DB-backed resolver tests use Postgres fresh DB `-p 1`.

## Real-code findings (from planning, grounded)
- **Secret resolution today** (runCustomHTTP worker.go:1813-1903): per header value, order is strict ‚Äî (1) `orgID,_ := w.cfg.Projects.OrgIDForProject(ctx, c.projectID)` (orgID ONLY from trusted run context, never input JSON); (2) `secretRefRe.ReplaceAllStringFunc` resolves `{{secret:NAME}}` ‚Üí `Secrets.Resolve(ctx, orgID, name)`, sets `secretBearing`; (3) `substituteVars` runs SECOND on the secret-resolved result ‚Äî so a `{{secret:X}}` arriving via a `{{name}}` upstream value is NEVER resolved (secret pass already done); (4) body forbids secrets entirely (post-substitute match ‚Üí `errRequestFailed`); (5) `secretBearing && !AllowResponseBody` ‚Üí store only `{"status":N}`. `secretRefRe = \{\{\s*secret:([A-Za-z0-9_\-]+)\s*\}\}`. **P2b boundary:** expr engine runs as a PEER to `substituteVars` (the `{{name}}` channel only), AFTER the secret pass; treats `secret:` as literal; touches none of `secretRefRe`/`Secrets.Resolve`/`OrgIDForProject`/`{status}` guard.
- **dependsOn for S-2:** `loadInputs` reads `SELECT depends_on FROM todos WHERE id=$1` (direct deps, `pq.StringArray`); planner `depSet` (planner.go:279-286) is also direct-deps-only. `loadInputs` only materializes direct deps' items. **A1 decision: direct-deps + project-scope suffices** (a transitive `$node` id would be a dead reference ‚Äî no items loaded). Cross-project defense: resolver loads each candidate dep's `project_id`, rejects any `!= c.projectID`.
- **Reuse vs parallel:** `substituteVars` (worker.go:1999) = the channel P2b PARALLELS (unchanged). `resolveOutputText`/`resolveVariables` = legacy seam, stays. `loadInputs`/`itemsForDep` (P2a) = items source (`ctx.Self`). No existing restricted-grammar engine ‚Äî `internal/expr` is net-new.
