# AI Studio (llm-agent-studio) — 6 维视觉 / 交互审计

**审计日期：** 2026-06-12
**基线：** 抽象 6 维标准 + 设计 token（`web/src/index.css`，逐字取自原型 :root）
**证据：** 14 页 × 桌面 1440 / 移动 390 全页截图（`/tmp/ui_audit/*.png`）+ 源码定位
**前提（不作为缺陷）：** 暗色单主题为产品设计语言；红色方块为 `PROVIDER=fake` 测试图，按内容忽略，仅评其布局/占位行为。

---

## 总评：**3.0 / 5**

桌面端是一套克制、专业、token 纪律严明的暗色后台：琥珀单主色用得节制（仅主 CTA + 选中态），制片轨道语义色（蓝/紫/琥珀/绿）落地清晰，字体层级与卡片样式跨页一致，登录/Prompt/存储/平台设置这类「单列表单」页几乎无可挑剔。**最大、也是全局性的弱点是响应式：移动端是「桌面布局塞进 390px」而非真正重排。** 资产库的筛选 aside 不折叠、所有数据表（成员/成本/平台用户/平台组织）在窄屏被裁切且无横滚提示、Badge 文案在挤压时逐字竖排、审核看板纯键盘操作在触屏不可用——这些让「移动端能用」从体验降到「勉强能看」。桌面拿 4 分，移动端拖到 2 分，综合 3.0。修复集中且廉价：绝大多数是「给容器加一个响应式断点 / 给 Badge 加 `whitespace-nowrap` / 给筛选栏套 Sheet」即可，不需要重做信息架构。

## 逐维评分表

### 1. 视觉层级 — **4 / 5**
- 标题（`text-[22px] font-bold font-heading`）与正文/说明（`text-[12px] text-text-3`）对比明确，每页主行动按钮（琥珀实心「新建项目 / 添加模型 / 保存」）是页内唯一高饱和元素，眼睛第一落点正确。证据：`10-projects-desktop`、`15/16/20`。
- 工作台顶栏「重新运行」（琥珀实心）vs「取消」（描边）主次分明，时间线节点用轨道色 + done/pending 徽章建立纵向阅读节奏。证据：`11-workbench-desktop`。
- 减分项：工作台右侧「选中工件」面板默认空（仅一个红块），却占满整列 ~1/3 宽度，desktop 上中列时间线内容窄、两侧空旷，焦点被稀释。建议未选中时给该列空态文案或收窄。`features/workflow/*`（WorkbenchPage 三栏 grid）。
- 减分项：任务中心进度用「方块点阵 + 百分比」表达，100% 行的 5 个琥珀块与「去审核」绿色按钮抢视觉，主次略乱。`12-tasks-desktop`。

### 2. 一致性 — **4 / 5**
- Badge / Button / 卡片 / 表头样式跨页统一，copy 语气一致（简短中文 + 行内说明），token 全程走 CSS 变量无硬编码色值。`components/studio/Badge.tsx`、`components/studio/Button.tsx`。
- 「按邮箱添加」行在成员页与平台设置页是同一模式，但两页移动端表现不一致：平台设置页 add-row 会换行（`20-platform-settings-mobile` 正常），成员页同结构却溢出裁切（`17-members-mobile`，「添加」按钮被切）。同一交互两套结果 = 一致性破绽。`features/members/MembersPage.tsx:135` vs 平台页对应行。
- 平台区左栏导航（平台/全部组织/用户管理 3 项）与业务区左栏（项目/任务…9 项）是两套 rail，切换时整条导航换内容，缺面包屑或区域标识，用户易迷失「我在平台层还是组织层」。`app/AppShell.tsx:99-201`。
- 状态徽章语义偶有重叠：`running` 与 `pending` 同为琥珀同款圆点（`Badge.tsx:12,14`），仅靠文案区分，扫读时两种状态视觉等价。

### 3. 布局与间距 — **3 / 5**
- 间距走统一 scale（`gap-6 p-6` 页级、`p-5/p-[18px]` 卡级），表单页留白节奏舒服。`16-storage-desktop`、`19-prompt-desktop`。
- 桌面端内容无 max-width 约束：成员/模型/存储/平台用户等页的表格与 header 横铺到 1440 满宽，`成员管理` 一行邮箱 + 角色 select 之间出现巨大空白（`17-members-desktop`），台账行同理（`18-cost-desktop` 列间距过疏）。建议给 `<main>` 内容区加 `max-w-[1200px]` 或表格列宽收敛。
- 资产库桌面：左 `w-56` 筛选 aside 合理，但右侧资产网格用 `minmax(150px,1fr)`，单资产时一个 150px 小卡孤零零贴左，右侧大片空，低数据态显空洞。`features/library/LibraryPage.tsx:101,175`。
- 工作台三栏在 desktop 比例失衡（左信息栏窄、中时间线窄、右工件栏宽且常空），中列大量竖向空白。`11-workbench-desktop`。

### 4. 排版 — **4 / 5**
- 类型 scale 克制（22/15/13/12/11px 几档），heading 用 Space Grotesk、正文 Noto Sans SC、数值/ID/时间用 JetBrains Mono 等宽并右对齐（`CostCenterPage.tsx:154,164,167` 金额 mono 右对齐），数字栏对齐干净。
- `text-3`（#666D78）用于次要说明文字，在 `bg-base`（#17191E）上对比约 3.3:1，低于 WCAG AA 正文 4.5:1（见维度 6）；纯排版上字号 11–12px + 低对比，移动端说明文字偏吃力。`index.css:80`。
- 减分项：移动端长英文/ID 不换行（平台组织 ID `font-mono` 被裁，`21-platform-orgs-mobile`），mono 列在窄屏不收缩。
- 行长在无 max-width 的宽桌面上偏长（说明段落横跨整屏），可读性略降。

### 5. 响应式 — **2 / 5**（全局最弱）
- **资产库筛选 aside 不折叠**：`LibraryPage.tsx:175` 的 `<aside className="flex w-56 flex-shrink-0 …">` 无移动断点，390px 下吃掉 224px，资产网格被压到 ~150px 半屏，标题「资产库 1个资产」挤作一团。`14-assets-mobile`。这是最严重单点缺陷——筛选栏应在 `<md` 收进 Sheet/抽屉，与 AppShell rail 同款处理。
- **数据表窄屏裁切无提示**：成员「移除」、成本表 `Provider·Model/类型/用量/金额`、平台用户 `平台管理员/组织数/操作(查看/删除)`、平台组织 `创建时间/成员数` 在 390px 全部被切到屏外（`17/18/22/21-mobile`）。底层 `ui/table.tsx:9` 已有 `overflow-x-auto`，所以是「能横滚但无滚动条/渐隐提示」的可发现性问题——平台用户页核心是行操作，操作列看不到 = 移动端功能性残废。
- **Badge 挤压逐字竖排**：`Badge.tsx:8` 缺 `whitespace-nowrap` / `shrink-0`，窄屏下「用服务端密钥」「已启用」（`15-mobile`）、「未配置」（`16-mobile`）被压成竖排单字。一处组件级 fix 全局收益。
- **成本 StatCard 固定三列**：`CostCenterPage.tsx:88,100` 硬编码 `grid grid-cols-3`，移动端三卡硬挤，`¥0.00` 溢出卡片。应 `grid-cols-1 sm:grid-cols-3`。
- **成员 add-row 固定横排**：`MembersPage.tsx:135` `<div className="flex gap-2">` 无 `flex-col sm:flex-row`，移动端「添加」按钮被切。
- 做得对的：AppShell rail→汉堡抽屉重排干净（`app/AppShell.tsx:230-307`）；工作台三栏在移动端正确改纵向堆叠（`11-workbench-mobile`）；登录/Prompt/存储/平台设置表单移动端宽度自适应良好。

### 6. 可达性与状态 — **3 / 5**
- 全局 focus-visible 有琥珀 2px 描边（`index.css:139-144`），`prefers-reduced-motion` 关停所有动画（`index.css:163-170`），徽章带圆点 + 文案双编码（非纯色区分，`Badge.tsx:41`），破坏性操作有二次确认弹窗（`MembersPage.tsx:232-261`）——基础盘扎实。
- 加载/空/错态覆盖完整：成员/成本/资产页均有 Skeleton + 错误重试 + 空态文案（`MembersPage.tsx:169-228`、`CostCenterPage.tsx:52-97`、`LibraryPage.tsx:82-100`）。
- **对比度不足**：`text-3`（#666D78）on `bg-base`（#17191E）≈ 3.3:1，低于 AA 4.5:1，而它被用于所有页的说明文字、时间戳、占位标签、事件日志（`11-workbench-desktop` 左栏事件日志整列偏灰难读）。`index.css:80`。
- **审核看板纯键盘操作**：`13-review` 顶部「←/→ 浏览 · A 采纳 R 退回 E 重生成」是唯一操作入口，触屏无对应按钮，移动端无法采纳/退回（`13-review-mobile`），且快捷键说明在窄屏换行成两段。`features/review/ReviewBoardPage.tsx`。
- **资产卡徽章溢出**：`待审核` 徽章绝对定位贴在 150px 缩略图上，桌面被卡宽裁切右半、移动端压在红块上（`14-assets-desktop/mobile`），状态可读性受损。`features/library/LibraryPage.tsx`（AssetCard 徽章定位）。
