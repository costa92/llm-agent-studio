# AI Studio Web（M5 前端 SPA）

驱动 studio 后端（studiod，M1–M4 接口表）的 React 19 单页应用 —— 登录、组织/项目管理、实时制片轨道（SSE 状态机）、剧本/分镜视图、HITL 审核看板、资产库、成本中心、模型配置、Prompt Builder。

**这是 Node/pnpm 工具链，不走 GOWORK**（与仓内 Go 命令隔离）。

## 栈

- **构建**：Vite 8 + React 19 + TypeScript 6
- **样式**：Tailwind v4（经 `@tailwindcss/vite` 插件，**无** `tailwind.config.js`，theme 写在 `src/index.css` 的 `@theme`）+ shadcn/ui（radix-nova preset）primitives + 自建 cva 组件（制片轨道/pip/血缘等）
- **路由**：`@tanstack/react-router` **file-based**（`@tanstack/router-plugin` + `@tanstack/router-cli` 的 `tsr generate` 生成 `src/routeTree.gen.ts`，gitignored）
- **服务端状态**：TanStack Query
- **SSE**：`@microsoft/fetch-event-source`（见下方「SSE 决策」）
- **表单**：react-hook-form + zod + `@hookform/resolvers`
- **测试**：Vitest + `@testing-library/react` + jsdom + `@testing-library/jest-dom`（mock `fetch` + 注入式 SSE client，**无真实后端、无浏览器**）

## 命令（在 `web/` 内运行）

```bash
pnpm install        # 装依赖（lockfile 已提交）
pnpm dev            # tsr generate + vite dev server（默认 :5173）
pnpm build          # tsr generate + tsc -b（类型检查）+ vite build → dist/
pnpm test           # tsr generate + vitest run（全量单测）
pnpm lint           # eslint .
pnpm preview        # 本地预览 dist/
```

> 所有脚本前置 `tsr generate`：`router-plugin` 只在 vite 阶段生成 `routeTree.gen.ts`，而 `tsc -b` 在它之前跑会因缺该文件失败 —— 故 build/test/dev 都先生成路由树。

## 开发代理（dev proxy → studiod :8083）

`vite.config.ts` 把 `/api/*` 转发到 `http://localhost:8083`（studiod 的 `HTTP_ADDR` 默认端口，见 `internal/config/config.go`）。dev 下 SPA 与 API 同源，**无 CORS 问题**。

本地联调时先起活后端（PG + `JWT_SECRET` + `HTTP_ADDR=:8083`），再 `pnpm dev`，浏览器访问 dev server 即可登录联调。

## Auth 机制（与 `llm-agent-authz` 对齐）

- **access token**：登录返回的 `access_token` 存在**内存**（模块级变量，**不进 localStorage，防 XSS**），每个 API 请求注入 `Authorization: Bearer <token>` 头。
- **refresh token**：httpOnly cookie `authz_refresh`（`Secure; SameSite=Strict; Path=/api/auth`），前端读不到、也不需要读。
- **refresh-on-401**：`apiFetch` 命中 `401` 时，single-flight 调一次 `POST /api/auth/refresh`（带请求头 `X-CSRF: 1` 双提交 + `credentials:"include"` 带 cookie），拿到新 access token 后重试原请求一次；刷新失败 → 清 token + 抛 `AuthError`。
- **登录/登出**：`POST /api/auth/login` body `{email,password}`（`credentials:"include"`）；`POST /api/auth/logout`（带 `X-CSRF: 1`）。
- **RBAC**：角色由后端 `RequireScopeRole`（scope_kind=`org`）强制。前端按角色**乐观显示 + 后端强制** —— 通过 admin-only 探针（`GET /api/orgs/{org}/model-configs` 的 `200`/`403`）推断当前用户在该 org 是否 admin，据此隐藏/显示审核·成本·模型配置入口。**这是 UX，不是安全边界**（详见 [docs/m5-deferred.md](../docs/m5-deferred.md)）。

## SSE 决策（用 `@microsoft/fetch-event-source`，不用原生 `EventSource`）

制片轨道实时流 `GET /api/projects/{id}/events/stream` 挂在 viewer+ 路由下，**必须带 `Authorization: Bearer` 头**。原生 `EventSource` 无法设置请求头（只能带 cookie），而 access token 不在 cookie 里 —— 故原生 `EventSource` 不可用。

结论：用 `@microsoft/fetch-event-source`（GET + `Authorization: Bearer` 头 + 自动重连 + `openWhenHidden:true`）。

**续传机制**：服务端 SSE（`sse.go`）不发 `id:` 字段、不读 `Last-Event-ID`，每次（重）连都从 `after=0` 全量回放历史事件。故客户端**不依赖** `Last-Event-ID`，防重复渲染靠按 `seq` 去重的制片轨道 reducer（`src/lib/timeline.ts`）—— 重连全量回放的旧帧被 seq-dedup 吞掉，再无缝接实时帧。

## 目录布局（feature-dir）

```
src/
├── index.css                 # @import "tailwindcss" + shadcn theme + 暗色 token 覆写
├── main.tsx                  # AuthProvider > QueryClientProvider > RouterProvider
├── routeTree.gen.ts          # GENERATED by tsr — gitignored
├── test/                     # jest-dom setup + mockFetch/注入式 SSE 帧 helpers
├── lib/                      # utils(cn)、types(后端线缆类型)、apiClient、sse、timeline
├── components/ui/            # shadcn primitives（lint 忽略 react-refresh）
├── components/studio/        # 自建 cva 组件（Badge/Button/TimelineStage/PipGroup/...）
├── app/                      # auth context、rbac、AppShell（IconRail 导航）
├── features/                 # projects / workflow / review / library / cost / prompt
└── routes/                   # TanStack file-based 路由（__root / login / _authed/*）
```
